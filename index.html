        Update this site

        GitHub Pages hosts this site, so publishing changes happens through GitHub instead of an on-site login.

        1. Open the repository in github.dev (press . on the repo) for a full browser editor. Edit the copy inside assets/js/content.js or add Markdown posts, then commit.
        2. Prefer static site generators? Use GitHub Pages with Jekyll or Quarto and a Pages GitHub Action to build and deploy, all within GitHub infrastructure.
        3. Draft rich posts locally with Editor.js and export to Markdown, then commit via GitHub for a safe, token-free workflow.

        Need to tweak copy quickly? Enable the local edit toolbar to prototype changes before committing.
