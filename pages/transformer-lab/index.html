Tokens Attention Pipeline

Change the sentence and every panel recomputes from scratch. The projections W_Q, W_K, W_V are deterministic random matrices so the same sentence always lands on the same attention — good for learning, not a trained model.

Read this first

  1. What it does A single transformer block turns a sequence of tokens into a sequence of context-aware vectors. Each token "looks at" every other token and pulls in the information it cares about.
  2. Try this Edit the sentence, then click through the four stages. Pick any token — the same token stays highlighted across Q/K/V, attention rows, and output.
  3. Look for this In the attention heatmap, bright cells tell you which tokens the row's query decided to blend. The output vector below is just that weighted sum.

Step 1 — break the sentence into tokens

Real models use subword tokenizers like BPE or WordPiece so rare words share prefixes. Here we use a simpler word-level split and wrap with [CLS] / [SEP] sentinels.

Click any token below — it stays selected across every stage so you can follow the same token's journey.

Step 2 — each token becomes three vectors

Every token's embedding e is multiplied by three fixed matrices to produce Q (query — what am I looking for?), K (key — what do I advertise?) and V (value — what do I carry?).

Q = e · WQ  ·  K = e · WK  ·  V = e · WV

Step 3 — every query scores every key

Score sij = Qi · Kj / √d. The softmax across each row turns those scores into weights that sum to 1 — those are the attention weights.

Hover or click a row to see which tokens that query ends up blending. Brighter cells = higher weight.

Step 4 — blend the value vectors

The context vector for token i is a weighted sum of the value vectors from every other token, using the attention weights from Step 3.

outputi = Σj αij · Vj
