In plain English — what is supervised learning?

You have a pile of flashcards: on one side an input (a photo, a sentence, a row of numbers), on the other an answer (a label, a score, a yes/no). Supervised learning is the problem of "study these flashcards so you can answer new ones you've never seen." The model is a function with knobs; training adjusts the knobs so the model's answers match the labels.

Regression = the answer is a number ("how much will this house sell for?"). Classification = the answer is a category ("spam or not?"). The three methods here are the small, interpretable, reliable versions of each — used every day for everything from medical risk scores to A/B tests to linear baselines for deep models.

Words that show up a lot:

  • Supervised learning — learning from labelled (x, y) pairs.
  • Regression — predict a real number.
  • Classification — predict a category.
  • Loss — the number we minimise during training. Squared error, cross-entropy, hinge.
  • MLE — "pick θ that makes the observed data most probable." A derivation technique, not a specific algorithm.
  • Regularisation — add a penalty on ‖θ‖ to the loss so the model doesn't over-fit. L2 = ridge; L1 = lasso (sparse).
  • Decision boundary — the surface in input space where a classifier switches its prediction.
  • Support vector — a training point that lies exactly on an SVM's margin. Only these determine the solution.

That's exactly the OLS objective. Pick a different noise distribution and you get a different loss: Laplace noise → L1 (absolute) loss; Bernoulli output → cross-entropy (next section); Poisson count → Poisson regression. Every classical loss is "−log p(data | θ) for some assumed distribution."

Shape watch. MLE → minimise −log likelihood. Add a prior p(θ) → minimise −log likelihood − log prior = MAP estimation. Gaussian prior → L2 regularisation (ridge). Laplace prior → L1 (lasso). The "loss" and the "regulariser" both come from the same probabilistic formalism.

4 · Support Vector Machine — widest margin

Kid version. Draw a line between the two classes. Pick the one with the biggest gap — the margin — on either side. Only the points that actually touch the margin matter. They're the support vectors.

5 · Kernels — SVMs beyond straight lines

Kid version. If the two classes aren't separable by a straight line, lift them into a higher-dimensional space where they are. The "kernel trick" lets you do that without ever writing down the high-dimensional coordinates.

The dual SVM only ever touches data through dot products . Replace that with a kernel and you get an SVM in whatever space φ maps to — without computing φ explicitly.

Linear

Original feature space. The demo above.

Polynomial

Implicitly adds all monomial interactions up to degree d.

RBF (Gaussian)

Implicitly infinite-dimensional. The go-to general-purpose kernel.

The same trick applies to kernel ridge regression, Gaussian processes (RBF => squared exponential covariance), and kernel PCA. Modern deep nets have mostly replaced kernels — they learn the feature map φ from data — but when you only have hundreds of samples, kernel SVM is still state-of-the-art.

6 · Generalisation — bias, variance, regularisation

Kid version. A model that memorises the training set perfectly (high variance) will fail on new data. A model too simple to capture the pattern (high bias) also fails. Regularisation tells the loss "keep the knobs small" to land somewhere in the middle.

The expected test error of a learned predictor decomposes into three terms:

  • Bias — systematic error from an inflexible model. Shrinks with capacity.
  • Variance — sensitivity to the training sample. Grows with capacity.
  • Irreducible — noise inherent to the data; no model can fix it.

Regularisation adds a penalty on the parameter norm:

Ridge · L2

  • Shrinks weights smoothly toward zero.
  • Closed form: θ̂ = (X⊤X + λI)−1X⊤y.
  • MAP with a Gaussian prior on θ.

Lasso · L1

  • Drives many coefficients to exactly zero — sparse solution.
  • Implicit feature selection.
  • MAP with a Laplace prior.

Elastic net

  • Hybrid of the two.
  • Sparse like lasso, stable like ridge.

Deep-learning regularisation adds more tools — dropout, batch norm, data augmentation, early stopping, weight decay (= L2) — all in service of the same bias-variance trade-off.
